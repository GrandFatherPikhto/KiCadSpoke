# `kicadspoke/geometry` – Геометрические утилиты

## Назначение

Модули в директории `geometry/` предоставляют низкоуровневые геометрические функции и классы, используемые для расчёта позиций компонентов, via и дорожек (tracks), построения ограничивающих областей (keepout), поиска свободного места, предсказания положения падов после перемещения/поворота, генерации сетки термовиа и преобразования локальных координат шаблонов в глобальные.

Эти модули **не зависят** от KiCad и адаптера – они оперируют только координатами и векторами, что делает их легко тестируемыми и переиспользуемыми. Они используются в основном в `placement/services/manual_position_calculator.py`, `placement/services/clone_position_calculator.py`, `via_planner.py` и других модулях расстановки.

---

## Структура

```
geometry/
├── __init__.py             # Экспорт публичных функций
├── keepout.py              # Ограничивающие прямоугольники и поиск свободного места
├── pad_projection.py       # Предсказание позиции пада (только для диагностики)
├── placement.py            # [УСТАРЕЛ] – оставлен для совместимости, будет удалён
├── spoke_layout.py         # Преобразование шаблона для ManualSpoke (БЕЗ треков)
├── clone_geometry.py       # Преобразование шаблона для ClonePlacement (с треками и mirror)
└── thermal_grid.py         # Генерация сетки термовиа
```

---

## Модули и их функции

### `keepout.py` – Ограничивающие области и поиск свободного места

**Назначение:**  
Определяет класс `Rect` (выровненный по осям прямоугольник) и предоставляет функции для построения keepout-областей из bounding box'ов, проверки точки на занятость (с учётом радиуса via) и поиска свободного места вокруг идеальной позиции (по спирали или вдоль прямой).

**Ключевые классы и функции:**

| Имя | Описание |
|-----|----------|
| `Rect` | AABB-прямоугольник. Конструктор: `Rect(min_x, min_y, max_x, max_y)`. Методы: `from_bbox(bbox, clearance)` – создаёт прямоугольник из `Box2` с запасом; `from_circle(center, radius)` – аппроксимирует окружность квадратом; `intersects(other)` – проверяет пересечение. |
| `point_is_clear(point, via_radius, keepout)` | Проверяет, свободна ли точка (окружность радиуса `via_radius` не пересекает ни один keepout-прямоугольник). |
| `build_keepout(bboxes, clearance_mm, mm_per_unit)` | Принимает список `Box2` (от адаптера) и строит список `Rect` с зазором `clearance_mm` с каждой стороны. Пропускает `None`-элементы. |
| `find_free_point(ideal, keepout, via_radius, preferred_direction=None, step_mm=0.1, max_radius_mm=3.0, n_directions=8)` | Ищет ближайшую свободную точку вокруг `ideal` расширяющимися кольцами. На каждом кольце сперва пробуется `preferred_direction` (если задано), затем `n_directions` точек равномерно по кругу. Возвращает `Vector2` или `None`. |
| `find_free_point_along_line(ideal, keepout, via_radius, line_direction, step_mm=0.1, max_radius_mm=3.0)` | Ищет свободную точку вдоль прямой, проходящей через `ideal`, с направлением `line_direction` (единичный вектор). Проверяет `ideal`, затем отступает в обе стороны с шагом. |

**Используется в:** `via_planner.py` **только** для поиска места под термовиа (единственный случай, где требуется автоматический поиск). Для всех остальных via (спицевых и компонентных) поиск не применяется – они размещаются строго по координатам шаблона.

### `pad_projection.py` – Предсказание позиции пада

**Назначение:**  
Предсказывает абсолютную позицию конкретного пада компонента, если компонент будет перемещён в новое место и повёрнут на заданный угол (с учётом возможного зеркалирования при флипе на обратную сторону). Логика централизована в этом модуле, чтобы избежать дублирования в разных частях проекта.

**Ключевые функции:**

| Имя | Описание |
|-----|----------|
| `local_pad_offset(fp, pad)` | Возвращает смещение пада относительно центра футпринта в **неповёрнутой** локальной системе координат (постоянная геометрия). |
| `predict_pad_position(fp, pad, dest, angle_deg, needs_flip)` | Предсказывает абсолютную позицию пада после перемещения футпринта в `dest` и поворота на `angle_deg` с учётом зеркалирования локального X при `needs_flip=True`. |

**Используется в:**  
В текущей версии KiCadSpoke эта функция **не используется** в основном коде, так как все via (включая GND via) вычисляются исключительно по геометрии шаблона, без обращения к живой плате. Однако модуль оставлен для диагностических скриптов (например, `test_pad_mirror_convention.py`) и для возможного использования в будущем (например, для ручной корректировки позиций или проверки коллизий).

### `spoke_layout.py` – Преобразование для ManualSpoke

**Назначение:**  
Преобразует локальные координаты шаблона спицы (`along`, `across`) в абсолютные координаты платы для `ManualSpoke` (привязка к паду IC) с учётом сдвига `(shift_x, shift_y)` и поворота `rotation_deg` конкретной спицы. Также вычисляет итоговый угол поворота компонента и генерирует все via (как на уровне спицы, так и на уровне компонента) в виде `ResolvedVia` с абсолютными координатами и цепью (если `net` не указан, используется `rule.net`).

**Важно:** в текущей реализации `ManualSpoke` **не поддерживает треки** – они присутствуют только в `ClonePlacement`. Это связано с тем, что ручные спицы (развязка FPGA) обычно не требуют клонирования дорожек.

**Ключевые классы и функции:**

| Имя | Описание |
|-----|----------|
| `rotate_local_offset(along_mm, across_mm, rotation_deg)` | Поворачивает локальный вектор `(along, across)` на заданный угол вокруг нуля (без переноса). |
| `local_to_absolute(origin, along_mm, across_mm, rotation_deg)` | Преобразует локальный вектор в абсолютную позицию относительно `origin` с учётом поворота. |
| `ResolvedVia` | Полностью разрешённая via: абсолютная позиция, цепь, параметры отверстия. |
| `ResolvedTrack` | Полностью разрешённый отрезок дорожки: начальная и конечная точки (абсолютные), ширина, цепь, абсолютный слой. **Используется только в ClonePlacement.** |
| `ComponentLayout` | Структура, описывающая размещение одного компонента: ref, роль, позиция, угол, список via. |
| `SpokeLayout` | Структура, описывающая все элементы спицы: origin, список via (уровня спицы), список компонентов, список треков. |
| `apply_spoke_geometry(pad_position, spoke, template, rule_net, role_to_ref)` | Основная функция – принимает позицию пада FPGA, данные спицы (`ManualSpoke`), шаблон (`SpokeTemplate`), цепь правила и сопоставление роль→ref. Возвращает `SpokeLayout` с абсолютными координатами всех элементов (кроме треков – они всегда пустые). |

**Используется в:** `manual_position_calculator.py` для расчёта позиций компонентов и via.

### `clone_geometry.py` – Преобразование для ClonePlacement

**Назначение:**  
Аналог `spoke_layout.py`, но для `ClonePlacement` (клонируемых размещений). Отличия:
- `origin` может быть задан абсолютными координатами `(origin_x_mm, origin_y_mm)` или как сдвиг от якоря (если задан `anchor_ref` или `anchor_role`).
- `net` каждой via и трека **резолвится через** `net_resolution.resolve_net()` с учётом `params` и `net_overrides` (нет дефолтной `rule_net`). Если `net` не задан – фатальная ошибка, так как нет разумного дефолта.
- Поддерживается привязка к якорному компоненту/паду через `anchor_ref`/`anchor_role` и `anchor_pad`.
- Поддерживается **зеркалирование** (`mirror=True`) – вся конструкция отражается по оси X, слои инвертируются, углы компонентов пересчитываются по формуле `180°−φ`.
- **Поддерживаются треки** – они преобразуются так же, как via и компоненты, с наследованием слоя от шаблона (если не задан явно).

**Ключевые классы и функции:**

| Имя | Описание |
|-----|----------|
| `_resolve_clone_via(origin, via, rotation_deg, clone, mirror)` | Преобразует `TemplateVia` в `ResolvedVia` с использованием `resolve_net` для цепи. Учитывает `mirror`. |
| `_resolve_clone_track(origin, track, rotation_deg, clone, tpl_layer, mirror)` | Преобразует `TemplateTrack` в `ResolvedTrack` с использованием `resolve_net` для цепи. Учитывает `mirror` и наследование слоя. |
| `_mirror_x(origin, p)` | X-зеркало точки относительно вертикальной оси через `origin`. |
| `apply_clone_geometry(clone, template, role_to_ref, anchor_position=None, mirror=False)` | Принимает `ClonePlacement`, шаблон и сопоставление роль→ref, а также опциональную абсолютную позицию якоря (если задан). Возвращает `SpokeLayout` с абсолютными координатами всех элементов (включая треки). |

**Используется в:** `clone_position_calculator.py` для расчёта позиций компонентов, via и треков для клонируемых секций.

### `thermal_grid.py` – Генерация сетки термовиа

**Назначение:**  
Вычисляет абсолютные координаты для массива термовиа под термопадом компонента (например, `IC1`). Учитывает размер пада, отступы от края, количество строк/столбцов и возможность шахматного порядка (stagger).

**Ключевые функции:**

| Имя | Описание |
|-----|----------|
| `get_pad_size(pad)` | Возвращает размеры медного слоя пада `(width, height)` в нанометрах. Бросает `GeometryError`, если медные слои отсутствуют. |
| `compute_thermal_via_grid(pad, rows, cols, margin_mm, stagger=False)` | Генерирует список абсолютных позиций для via, размещённых равномерно внутри площадки с учётом отступов `margin_mm`. Параметр `stagger` включает шахматный порядок. Возвращает список `Vector2`. |

**Используется в:** `via_planner.py` для планирования термовиа.

---

### `placement.py` – **УСТАРЕЛ**

Содержит старую функцию `compute_position`, которая использовалась для автоматического размещения компонентов относительно границы зоны (`boundary`). После перехода на ручную стратегию (шаблоны) этот файл больше не используется и оставлен только для обратной совместимости со старыми тестами. В ближайшее время будет удалён.

## Примеры использования

### 1. Построение keepout и поиск свободной точки (для термовиа)

```python
from kicadspoke.geometry.keepout import build_keepout, find_free_point
from kipy.geometry import Vector2

# Получаем bounding box'ы падов через адаптер
bboxes = adapter.get_bounding_boxes(pads)
keepout = build_keepout(bboxes, clearance_mm=0.2)

ideal = Vector2.from_xy(10_000_000, 20_000_000)
via_radius = 0.3 * MM  # 0.3 мм в нанометрах

free_point = find_free_point(ideal, keepout, via_radius, preferred_direction=(1, 0))
if free_point is None:
    print("Не удалось найти свободное место")
else:
    print(f"Свободная точка: ({free_point.x/MM:.3f}, {free_point.y/MM:.3f}) мм")
```

### 2. Генерация сетки термовиа

```python
from kicadspoke.geometry.thermal_grid import compute_thermal_via_grid
from kicadspoke.kicad.adapter import KiCadBoardAdapter

adapter = KiCadBoardAdapter()
adapter.refresh_board()
fp = adapter.get_footprint("IC1")
pad = adapter.get_pad_by_number(fp, "145")

points = compute_thermal_via_grid(pad, rows=4, cols=4, margin_mm=0.5, stagger=False)
for p in points:
    print(f"Via at ({p.x/MM:.3f}, {p.y/MM:.3f}) мм")
```

### 3. Преобразование шаблона для клонирования (с треками и mirror)

```python
from kicadspoke.geometry.clone_geometry import apply_clone_geometry
from kicadspoke.config import load_config

cfg = load_config("config.yaml")
template = cfg.templates["pi_filter_4"]
clone = cfg.clone_placements[0]
role_to_ref = {"PI_FILTER_C1": "C601", "PI_FILTER_FB": "FB601"}

# Без зеркалирования
layout = apply_clone_geometry(clone, template, role_to_ref, mirror=False)
for via in layout.vias:
    print(f"Via: ({via.position.x/MM:.3f}, {via.position.y/MM:.3f}) мм, net={via.net}")
for track in layout.tracks:
    print(f"Track: ({track.start.x/MM:.3f}, {track.start.y/MM:.3f}) -> "
          f"({track.end.x/MM:.3f}, {track.end.y/MM:.3f}), net={track.net}")

# С зеркалированием (вся конструкция переворачивается на обратную сторону)
layout_mirrored = apply_clone_geometry(clone, template, role_to_ref, mirror=True)
# Углы компонентов пересчитаны по формуле 180°−φ, слои инвертированы
```

## Примечания

- Все координаты и размеры ожидаются в **нанометрах**, если не указано иное (поля `_mm` принимают миллиметры и конвертируются с помощью `MM = 1_000_000` из `utils/units.py`).
- `find_free_point` и `find_free_point_along_line` сейчас используются **только** для термовиа – все остальные via размещаются строго по заданным координатам без поиска.
- Модули `spoke_layout.py` и `clone_geometry.py` полностью независимы от зоны и `boundary` – вся геометрия задаётся в локальной системе шаблона, что делает конфигурацию поворотоинвариантной и предсказуемой.
- Для `ClonePlacement` цепи via и треков **обязательно** должны быть заданы (либо через `net` в шаблоне, либо через `nets`/`net_overrides` в клоне), так как нет `rule.net` по умолчанию. В противном случае выбрасывается `ValidationError`.
- Модуль `pad_projection.py` сохраняется для диагностических целей и возможных будущих расширений, хотя в основном потоке не используется.
- **Треки поддерживаются только в `ClonePlacement`.** В `ManualSpoke` треки не планируются (это архитектурное решение).

## Тестирование

Геометрические модули покрыты модульными тестами (в `tests/test_*.py`), которые проверяют:
- Корректность преобразования локальных координат в абсолютные.
- Построение keepout и поиск свободных точек.
- Генерацию сетки термовиа.
- Предсказание позиции пада (включая эмпирическую проверку конвенции зеркалирования).
- Преобразование треков в `clone_geometry.py` (поворот, зеркалирование, наследование слоя).

Для ручной проверки конвенции зеркалирования используется диагностический скрипт `test_pad_mirror_convention.py`, который выполняет реальный флип и поворот компонента в KiCad и сравнивает предсказания с фактическим положением пада.
