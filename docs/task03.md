# Многокритериальная параметрическая оптимизация топологии цепей питания (PDN) микросхем высокой плотности

## Введение

В современном проектировании высокоскоростных цифровых устройств критически важной задачей является обеспечение целостности питания (Power Integrity, PI) микросхем с высокой плотностью выводов (таких как FPGA/BGA). Стабильность работы кремниевого кристалла напрямую зависит от паразитной индуктивности и сопротивления цепей питания (PDN), что требует размещения каскадов блокировочных конденсаторов в непосредственной близости от выводов микросхемы.

![Сглаживающая цепь питания на примере 10CL006YE144C8G](images/fig01.png)

С ростом плотности компоновки и увеличением количества разнородных питающих напряжений ($\text{VCC\_INT}$, $\text{VCCIO}$, $\text{VCCD\_PLL}$) традиционная ручная трассировка топологии неизбежно приводит к пространственным конфликтам («толчее» компонентов и переходных отверстий). Для автоматизации этого процесса и поиска глобально оптимального расположения элементов разработана математическая модель, в основе которой лежит задача геометрической оптимизации связных графов в условиях нелинейных ограничений-неравенств.

![Несколько «спиц» питания](images/fig02.png)
![Via `+1V2_VCCINT сместилась вправо](images/fig03.png)

В рамках данной модели каждая «спица» питания (каскад из конденсаторов разной ёмкости, переходных отверстий и выводов FPGA) представляется в виде ориентированного графа с заданными приоритетами (весами пружин) для каждого ребра. Целевая функция алгоритма направлена на минимизацию суммарной длины критических проводников. При этом технологические зазоры (DRC) и фиксированные размеры посадочных мест компонентов формируют систему жестких нелинейных ограничений, основанных на евклидовых расстояниях. Разработанный подход позволяет автоматически синтезировать симметричную, компактную и помехозащищенную топологию печатной платы с учётом специфики и физических ограничений каждой конкретной питающей цепи.

## Описание базовой топологии «спицы» (Spoke)

Каждая индивидуальная спица питания реализует классический каскадный фильтр:

```
[Входящая линия питания] 
       │
       ▼
┌──────────────┐      ┌──────────────┐      ┌─────────────┐      ┌───────────┐
│ «Тяжёлый»    │ ───> │ «Лёгкий»     │ ───> │ Power Via   │ ───> │ Пад пина  │
│ конд. 4.7 uF │      │ конд. 100 nF │      │ (Слой B.Cu) │      │ FPGA      │
└──────────────┘      └──────────────┘      └─────────────┘      └───────────┘
       │                     │                     │              (Слой F.Cu)
       ▼                     ▼                     ▼
┌──────────────┐      ┌──────────────┐      ┌─────────────┐
│  GND Pad 2   │      │  GND Pad 2   │      │   GND Via   │
└──────────────┘      └──────────────┘      └─────────────┘
       │                     │                     │
       └─────────────────────┴─────────────────────┴─> [Опорный слой Земли]
```

- **Каскад емкостей**: На входе спицы (снаружи дворика FPGA) стоит тяжёлый конденсатор 4.7 мкФ, работающий как локальный резервуар энергии для сглаживания низкочастотных просадок. Ближе к выводу FPGA (внутри дворика) располагается быстрый конденсатор 100 нФ для фильтрации высокочастотных помех. Оба компонента монтируются на обратной стороне платы (слой B.Cu) прямо под выводами микросхемы для минимизации индуктивности проводников.
- **Трассировка и Via**: Линия питания проходит через пады конденсаторов, уходит сквозь плату с помощью переходного отверстия питания (Power Via) на верхний слой (F.Cu) и отдаётся целевому паду FPGA с контролируемым локальным уменьшением («горловиной») толщины дорожки.
- **Заземление**: Каждый конденсатор имеет отдельный земляной пад, который соединяется с опорным слоем через собственную GND Via. Петля возвратного тока должна быть минимальной.

При последовательном размещении серии таких спиц под корпусами типа EQFP-144 (шаг пинов $P_{pitch} = 0.5 \text{ мм}$) их физические габариты (ширина пада 0402 $\approx 0.6 \text{ мм}$, диаметр Via $\approx 0.5 \text{ мм}$) делают строго параллельную разводку невозможной ($0.6 \text{ мм} + \text{зазор} > 0.5 \text{ мм}$). Элементы вынуждены смещаться, образуя «толчею» и нарушая геометрию соседних трасс.

## Формальная математическая модель оптимизации

Задача решается глобально (одновременно) для всего массива спиц $N$ с использованием гладкой целевой функции и точной матрицы ограничений.

### 1. Геометрические константы

Определяются технологией производства и размерами компонентов:

| Символ | Описание | Типовое значение (пример) |
|--------|----------|---------------------------|
| $P_{pitch}$ | Шаг пинов FPGA | $0.5 \text{ мм}$ |
| $R_{via}$ | Радиус Power Via (с пояском) | $0.25 \text{ мм}$ |
| $R_{gnd}$ | Радиус GND Via (с пояском) | $0.25 \text{ мм}$ |
| $R_{fpga}$ | Радиус пада FPGA | $0.2 \text{ мм}$ |
| $R_{cap}$ | Радиус пада конденсатора (0402) | $0.3 \text{ мм}$ |
| $L_{comp}$ | Расстояние между центрами падов конденсатора (длина компонента) | $1.0 \text{ мм}$ |
| $S$ | Минимальный технологический зазор (Clearance) | $0.15 \text{ мм}$ |
| $Y_{min}$ | Минимальный отступ от линии падов FPGA до центра ближайшего элемента | $R_{fpga} + R_{cap} + S$ |

### 2. Единый вектор оптимизируемых переменных $\mathbf{X}$

Каждая спица $i$ описывается **12-ю вещественными переменными** (координаты центров элементов на плоскости):

$$
\mathbf{x}_i = \big[ x_{v,i},\, y_{v,i},\, x_{g,i},\, y_{g,i},\, 
x_{c1,i},\, y_{c1,i},\, x_{c1g,i},\, y_{c1g,i},\,
x_{c2,i},\, y_{c2,i},\, x_{c2g,i},\, y_{c2g,i} \big]
$$

Где:
- $V_i = (x_{v,i}, y_{v,i})$ — центр Power Via.
- $G_i = (x_{g,i}, y_{g,i})$ — центр GND Via.
- $C1_i = (x_{c1,i}, y_{c1,i})$ — центр питающего пада малого конденсатора (100 нФ).
- $C1G_i = (x_{c1g,i}, y_{c1g,i})$ — центр земляного пада малого конденсатора.
- $C2_i = (x_{c2,i}, y_{c2,i})$ — центр питающего пада тяжёлого конденсатора (4.7 мкФ).
- $C2G_i = (x_{c2g,i}, y_{c2g,i})$ — центр земляного пада тяжёлого конденсатора.

Общий вектор $\mathbf{X} = [\mathbf{x}_1, \mathbf{x}_2, \dots, \mathbf{x}_N]^T \in \mathbb{R}^{12N}$.

Координаты целевых падов FPGA $F_i = (x_{f,i}, y_{f,i})$ жёстко заданы геометрией корпуса:  
$x_{f,i} = i \cdot P_{pitch}$, $y_{f,i} = 0$ (линия падов принята за ось X).

### 3. Классификация цепей и направления развития

Каждому индексу $i$ сопоставлен тип питания $T_i \in \{1,2,3,4\}$:

- $T_i = 1$ (VCC_INT, ядро) — внутрь ($y > 0$), высший приоритет.
- $T_i = 2$ (VCCD_PLL) — внутрь ($y > 0$), высший приоритет.
- $T_i = 3$ (VCCA, аналог) — наружу ($y < 0$), средний приоритет.
- $T_i = 4$ (VCCIO, периферия) — наружу ($y < 0$), низший приоритет.

Функция направления $s_i$:
$$
s_i = \begin{cases}
+1, & T_i \in \{1,2\} \\
-1, & T_i \in \{3,4\}
\end{cases}
$$

### 4. Целевая функция $\mathcal{F}(\mathbf{X})$ (гладкая, квадратичная)

Минимизируется сумма энергий трассировки, замкнутой GND-петли, мягкой коллинеарности и штрафа за несоответствие пад конденсатора его земляному паду (равенство расстояния $L_{comp}$ обеспечивается отдельными ограничениями, но для устойчивости можно добавить слабый штраф, хотя в данном случае используем жёсткие ограничения-равенства).

$$
\mathcal{F}(\mathbf{X}) = \sum_{i=1}^{N} \Big( E_{\text{route}, i} + E_{\text{gnd\_loop}, i} + E_{\text{collinear}, i} \Big) \to \min
$$

#### а) Энергия трассировки линии питания
$$
E_{\text{route}, i} = w_{fv}(T_i) \cdot \Vert F_i - V_i \Vert^2 + w_{vc}(T_i) \cdot \Vert V_i - C1_i \Vert^2 + w_{cc}(T_i) \cdot \Vert C1_i - C2_i \Vert^2
$$
Веса $w$ задают иерархию приоритетов:  
$w_{fv} \gg w_{vc} \gg w_{cc}$ (например, 1000 : 10 : 1) и зависят от $T_i$ (более критичные цепи получают большие веса).

#### б) Энергия GND-петли (возвратный ток)
Притягивает GND Via к земляному паду малого конденсатора (самый высокочастотный контур):
$$
E_{\text{gnd\_loop}, i} = w_{gnd} \cdot \Vert G_i - C1G_i \Vert^2
$$
Дополнительно можно добавить притяжение GND Via к земляному паду тяжёлого конденсатора с меньшим весом, но для упрощения оставляем только один член.

#### в) Мягкая коллинеарность (штраф за отклонение от вертикальной оси пада FPGA)
Позволяет элементам смещаться по X, но штрафует за потерю «красоты»:
$$
E_{\text{collinear}, i} = w_{coll} \cdot \Big[ (x_{v,i} - x_{f,i})^2 + (x_{c1,i} - x_{f,i})^2 + (x_{c2,i} - x_{f,i})^2 + (x_{g,i} - x_{f,i})^2 \Big]
$$
Здесь $w_{coll}$ выбирается достаточно малым (например, 0.1), чтобы не доминировать над физическими приоритетами.

### 5. Система ограничений (неравенства и равенства)

Все ограничения должны быть дифференцируемыми (гладкими).

#### 5.1. Жёсткая связь падов конденсатора (равенство расстояния и ориентации)
Для каждого конденсатора задаём, что его питающий и земляной пады разнесены на длину $L_{comp}$ по горизонтали (ось X), т.е. компонент ориентирован горизонтально. Это упрощает модель и соответствует типовой компоновке (пады слева/справа):
$$
h_{\text{align},1,i} = (x_{c1,i} - x_{c1g,i})^2 - L_{comp}^2 = 0, \quad
h_{\text{align},2,i} = (y_{c1,i} - y_{c1g,i}) = 0
$$
Аналогично для $C2$:
$$
h_{\text{align},3,i} = (x_{c2,i} - x_{c2g,i})^2 - L_{comp}^2 = 0, \quad
h_{\text{align},4,i} = (y_{c2,i} - y_{c2g,i}) = 0
$$

#### 5.2. Физическая последовательность элементов вдоль оси Y
В зависимости от направления $s_i$ задаём линейные неравенства, обеспечивающие порядок:
$$
\begin{cases}
y_{v,i} < y_{c1,i} < y_{c2,i}, & s_i = +1 \quad(\text{внутрь}) \\
y_{v,i} > y_{c1,i} > y_{c2,i}, & s_i = -1 \quad(\text{наружу})
\end{cases}
$$
В виде неравенств (для $s_i=+1$):  
$h_{\text{seq1},i} = y_{c1,i} - y_{v,i} \ge 0$,  
$h_{\text{seq2},i} = y_{c2,i} - y_{c1,i} \ge 0$.  
Для $s_i=-1$:  
$h_{\text{seq1},i} = y_{v,i} - y_{c1,i} \ge 0$,  
$h_{\text{seq2},i} = y_{c1,i} - y_{c2,i} \ge 0$.

#### 5.3. Отступ от линии падов FPGA (знаковый барьер)
Все элементы спицы должны находиться по правильную сторону от $Y=0$ и не ближе $Y_{min}$:
$$
\begin{cases}
y_{*,i} \ge Y_{min}, & s_i = +1 \\
y_{*,i} \le -Y_{min}, & s_i = -1
\end{cases}
$$
для всех переменных $y_{*,i}$ (via, пады конденсаторов, включая GND Via и земляные пады).

#### 5.4. DRC-ограничения (непересечение объектов)

Все расстояния между любыми двумя объектами (центрами) должны быть не меньше суммы их радиусов плюс зазор $S$. Радиусы заданы: для Power Via $R_{via}$, для GND Via $R_{gnd}$, для падов конденсаторов $R_{cap}$, для падов FPGA $R_{fpga}$.

Обозначим множество всех центров объектов для спицы $i$:  
$\mathcal{O}_i = \{ V_i, G_i, C1_i, C1G_i, C2_i, C2G_i \}$ и фиксированные $F_i$.

Для каждой пары объектов $A \in \mathcal{O}_i \cup \{F_i\}$, $B \in \mathcal{O}_j \cup \{F_j\}$ (кроме случаев, когда это один и тот же физический объект, или когда это пары питающий-земляной пад одного конденсатора, которые связаны равенством, но DRC между ними не требуется) необходимо выполнить:
$$
\Vert A - B \Vert^2 - (R_A + R_B + S)^2 \ge 0
$$
где $R_A$ — соответствующий радиус (для падов FPGA используем $R_{fpga}$, для питающих и земляных падов конденсаторов — $R_{cap}$).

Особые случаи, которые **исключаются** из проверки (разрешённые касания/перекрытия в пределах одной спицы):
- Пара $V_i$ и $C1_i$ (они соединены дорожкой, но расстояние между ними должно быть > $R_{via}+R_{cap}+S$, это уже учтено в общем правиле, но если они очень близко, то DRC может быть нарушен, поэтому проверяем; исключаем только те, что должны быть соединены, но они всё равно должны соблюдать зазор, так что проверка остаётся).
- На самом деле, все объекты должны быть разнесены, включая соединённые, потому что они не перекрываются, а просто соединяются дорожкой, поэтому проверка для всех пар обязательна.

Однако нужно исключить проверку между питающим и земляным падом **одного и того же** конденсатора, потому что они являются частями одного компонента и их расстояние фиксировано $L_{comp}$, которое заведомо больше $2R_{cap}+S$ (обычно). Поэтому такие пары пропускаем.

Также следует исключить проверку между объектами, которые являются одним и тем же (например, $V_i$ с самим собой).

#### 5.5. Дополнительное ограничение на расстояние между Power Via и GND Via (для уменьшения паразитной индуктивности)
Можно добавить мягкий штраф в целевую функцию, но также можно ввести верхнюю границу, чтобы они не разбегались слишком далеко. Однако в целевой функции уже есть $E_{\text{gnd\_loop}}$, которая притягивает GND Via к земляному паду малого конденсатора, а этот пад в свою очередь связан с питающим падом, так что косвенно расстояние между V и G будет минимизироваться. Можно оставить без дополнительного жёсткого ограничения.

### 6. Аналитический Якобиан ограничений

Для эффективной работы SQP/SLSQP все ограничения должны иметь аналитические производные. Большинство ограничений имеют вид $h(\mathbf{X}) = \Vert A - B \Vert^2 - D_{safe}^2 \ge 0$ или линейные неравенства.

Для квадратичного ограничения:
$$
h_k = (x_A - x_B)^2 + (y_A - y_B)^2 - D_{safe}^2 \ge 0
$$
частные производные:
$$
\frac{\partial h_k}{\partial x_A} = 2(x_A - x_B), \quad
\frac{\partial h_k}{\partial y_A} = 2(y_A - y_B),
$$
$$
\frac{\partial h_k}{\partial x_B} = -2(x_A - x_B), \quad
\frac{\partial h_k}{\partial y_B} = -2(y_A - y_B).
$$

Для ограничений-равенств типа $(x_{c1} - x_{c1g})^2 - L_{comp}^2 = 0$:
$$
\frac{\partial h}{\partial x_{c1}} = 2(x_{c1} - x_{c1g}), \quad
\frac{\partial h}{\partial x_{c1g}} = -2(x_{c1} - x_{c1g}),
$$
и аналогично для Y-координат (если равенство по Y, то производная равна 1 или -1 для линейного).

Все производные легко вычисляются и формируют разреженную матрицу Якоби.

## Практическая реализация

Модель может быть реализована с помощью языка Python и библиотеки `scipy.optimize.minimize` с методом `'SLSQP'` или `'trust-constr'`. Алгоритм:

1. Задать геометрические константы и массив типов $T_i$ для каждого пина FPGA.
2. Сформировать начальное приближение $\mathbf{X}_0$ (например, все элементы поместить на линии $Y=0$ с небольшим случайным смещением).
3. Задать функцию вычисления $\mathcal{F}(\mathbf{X})$ и её градиент (можно аналитически или через автоматическое дифференцирование, но аналитический градиент несложен).
4. Задать функцию ограничений, возвращающую массив значений $h(\mathbf{X})$ (неравенства и равенства), а также функцию Якобиана (разреженную или плотную).
5. Запустить оптимизатор и получить оптимальные координаты.
6. Извлечь координаты и сгенерировать топологию.

Такой подход позволяет полностью автоматизировать размещение 40+ конденсаторов под FPGA, сократив время проектирования с часов до секунд.

## Заключение

Представленная математическая модель учитывает ключевые физические и технологические ограничения, обеспечивает глобальный оптимум и даёт гладкую, дифференцируемую задачу, пригодную для стандартных решателей. Внесённые дополнения (последовательность элементов, замкнутая GND-петля, DRC для GND Via, честные диагональные расстояния и мягкая коллинеарность) делают её полностью пригодной для практического применения в САПР печатных плат.
