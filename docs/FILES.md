## 🔹 Корневой каталог

### `placer.py`  
**Главный CLI-скрипт**  
- **Устройство:** модуль с парсером аргументов (`argparse`) и двумя командами: `apply` (применить расстановку) и `generate` (сгенерировать YAML-правила). Основные функции: `cmd_apply()`, `cmd_generate()`, `setup_logging()`.  
- **Что делает:** запускает весь процесс – загружает конфиг, инициализирует адаптер, планировщик, исполнитель. В режиме `--dry-run` только выводит план. Для генерации правил вызывает `RulesGenerator`.  
- **Способ:** использует стандартную библиотеку (`argparse`, `logging`, `sys`, `pathlib`), импортирует модули из пакета `decap_placer`. Не обращается напрямую к файлам кроме тех, что переданы аргументами (конфиг, .net, .pcb). IPC – через адаптер.

---

### `decap_placement.yaml`  
**Пример конфигурационного файла**  
- **Устройство:** YAML-документ с ключами `target_ref`, `boundary_zone`, `side`, `rotation_mode`, `via`, `thermal_via_array`, `rules`.  
- **Что делает:** задаёт все параметры расстановки (целевой компонент, зону, правила для каждого конденсатора).  
- **Способ:** читается функцией `load_config()` из `config.py` через `yaml.safe_load()` (библиотека `pyyaml`).  

---

### `decap_placer.txt`  
**Служебный файл со списком всех модулей** – не участвует в работе программы.

---

### `README.md`  
**Документация** – описание установки, использования, формата конфига и архитектуры. Не исполняемый код.

## 🔹 Пакет `decap_placer`

### `__init__.py`  
Пустой инициализатор – делает папку пакетом.

---

### `config.py`  
**Модели данных и загрузка конфигурации**  
- **Устройство:** dataclass’ы: `ViaConfig`, `PowerViaConfig`, `ThermalViaArrayConfig`, `SpokeComponent`, `Spoke`, `Rule`, `Config`. Функции: `_load_via_config`, `_load_power_via_config`, `load_config()`, а также вспомогательная `resolve_power_pin_facing()`.  
- **Что делает:** определяет структуру конфига и загружает YAML-файл в объекты Python.  
- **Способ:** использует `yaml` для чтения файла, `dataclasses` для моделей. Логирование – через `logging`.  

---

### `exceptions.py`  
**Пользовательские исключения**  
- **Устройство:** классы `PlacerError` (базовый), `BoardNotFoundError`, `ComponentNotFoundError`, `GeometryError`.  
- **Что делает:** стандартизирует обработку ошибок в проекте.  
- **Способ:** наследуются от `Exception`, используются в модулях для выбрасывания и перехвата.

## 🔹 Подпакет `geometry`

### `boundary.py`  
**Работа с полигоном зоны**  
- **Устройство:** функции: `polyline_points()`, `_ray_segment_intersection()`, `ray_boundary_distance()`, `polygon_signed_area()`, `closest_point_on_polygon()`.  
- **Что делает:** вычисляет пересечение луча с границей, знаковую площадь полигона и ближайшую точку на границе к заданной точке с нормалью наружу.  
- **Способ:** чистая математика (векторная алгебра), без внешних API. Координаты в нанометрах.

---

### `keepout.py`  
**Области, занятые падами, и поиск свободной точки**  
- **Устройство:** класс `Rect` (AABB), функции: `build_keepout()` (строит список прямоугольников из bounding box’ов), `point_is_clear()`, `find_free_point()` (поиск по расширяющимся кольцам).  
- **Что делает:** строит keepout-прямоугольники для падов компонентов и ищет ближайшее свободное место для via.  
- **Способ:** использует `kipy.geometry.Vector2` для координат, математические расчёты. Bounding box’ы получает через адаптер (вызов `adapter.get_bounding_boxes`).

---

### `relax.py`  
**Раздвижка точек вдоль ряда**  
- **Устройство:** функции `get_tangential_axis()`, `relax_1d()` (основной алгоритм кластеризации), `relax_positions()` (обёртка для 2D).  
- **Что делает:** раздвигает конфликтующие конденсаторы вдоль одной оси, чтобы выдержать минимальный зазор.  
- **Способ:** чистая математика, сортировка, группировка. Работает с координатами в нанометрах.

---

### `strategies.py`  
**Стратегии размещения**  
- **Устройство:** абстрактный класс `PlacementStrategy` и его реализации: `RadialStrategy`, `OrthogonalStrategy`, `FixedStrategy`, `BoundaryStrategy`.  
- **Что делает:** вычисляет позицию конденсатора относительно центра целевого компонента, вывода и границы зоны, используя заданный режим (`rotation_mode`).  
- **Способ:** использует функции из `boundary.py` (`ray_boundary_distance`, `closest_point_on_polygon`), а также `math` и `Vector2`.

---

### `thermal_grid.py`  
**Расчёт сетки термовиа**  
- **Устройство:** функции `get_pad_size()`, `compute_thermal_via_grid()` (создаёт список позиций для via внутри площадки).  
- **Что делает:** генерирует равномерную сетку (или шахматную) на термопаде с заданным отступом от краёв.  
- **Способ:** использует `kipy.board_types.Pad` для получения размеров и ориентации, преобразует локальные координаты в глобальные через поворот и смещение. Координаты – в нанометрах.

---

## 🔹 Подпакет `kicad`

### `adapter.py`  
**Адаптер для взаимодействия с KiCad через IPC**  
- **Устройство:** класс `KiCadBoardAdapter`, который инкапсулирует вызовы `kipy`. Методы: `refresh_board()`, `get_footprint()`, `get_footprints()`, `get_footprint_pads()`, `get_pad_by_number()`, `get_zone_by_name()`, `get_net_by_name()`, `get_bounding_boxes()`, `begin_commit()`, `push_commit()`, `drop_commit()`, `update_items()`, `create_items()`, `flip_selected()`, `commit_with_retry()`, `create_via()`.  
- **Что делает:** обеспечивает высокоуровневый доступ к плате KiCad: поиск компонентов, зон, цепей, получение bounding box’ов, выполнение транзакций, флип, создание via.  
- **Способ:** использует библиотеку `kipy` для IPC-коммуникации с процессом KiCad. Все операции отправляются по gRPC (через `kipy.KiCad`). Координаты и размеры передаются в нанометрах.

## 🔹 Подпакет `placement`

### `collision.py`  
**Проверка коллизий между компонентами**  
- **Устройство:** функции: `_radius_from_bbox()`, `compute_radii()`, `footprints_overlap()`, `check_collisions()`.  
- **Что делает:** вычисляет радиусы (по половине диагонали bounding box) для всех футпринтов и проверяет пересечения кругов-приближений с заданным зазором.  
- **Способ:** запрашивает bounding box’ы через адаптер (`adapter.get_bounding_boxes`) и использует математику для проверки расстояний.

---

### `executor.py`  
**Исполнитель команд (применение изменений в KiCad)**  
- **Устройство:** класс `BatchExecutor` с методом `execute()`, принимающим списки `MoveCommand` и `ViaCommand`. Внутри – вспомогательные методы `_needs_flip()`, `_flip_in_batches()`.  
- **Что делает:** разбивает команды на батчи, выполняет флип (при необходимости), затем обновляет позиции/углы компонентов и создаёт via. Использует транзакции с повторными попытками.  
- **Способ:** использует адаптер (`KiCadBoardAdapter`) для всех операций с платой. Для флипа вызывает GUI-action через `adapter.flip_selected`. Для транзакций – `commit_with_retry`.

---

### `planner.py`  
**Планировщик расстановки**  
- **Устройство:** класс `PlacementPlanner` с методами `plan_moves()`, `plan_vias()`, `plan()` и множеством вспомогательных (`_create_strategy`, `_get_boundary_polygon`, `_find_pad`, `_resolve_facing_angle`, `_merge_via_config`, `_plan_stitching_vias`, `_zone_preferred_direction`, `_build_via_keepout`, `_plan_thermal_vias`).  
- **Что делает:** вычисляет позиции и углы конденсаторов по правилам, выполняет раздвижку, строит keepout, планирует via (stitching и термовиа) с поиском свободных точек.  
- **Способ:** использует адаптер для получения данных о плате, стратегии для расчёта позиций, `relax_positions` для раздвижки, `keepout.find_free_point` для via. Работает с координатами в нанометрах и углами в градусах через `kipy.Angle`.

---

## 🔹 Подпакет `rules`

### `generator.py`  
**Генератор YAML-правил из .net и .kicad_pcb**  
- **Устройство:** класс `RulesGenerator` с методами `_pins_sorted_by_angle()`, `_filter_pins_min_spacing()`, `generate()`, `generate_yaml()`.  
- **Что делает:** парсит файлы цепи и платы, извлекает пины целевого компонента, распределяет конденсаторы по пинам (round‑robin с фильтром минимального расстояния), формирует структуру `Rule`/`Spoke` и возвращает YAML.  
- **Способ:** использует `parser.parse_net_file()` и `parser.parse_pcb_file()` для чтения S-выражений, затем собирает данные.

---

### `parser.py`  
**Парсеры S-выражений для .net и .kicad_pcb**  
- **Устройство:** функции `tag()`, `find_all()`, `find_first()`, `get_str()`, `parse_net_file()`, `parse_pcb_file()`.  
- **Что делает:** читает файлы .net и .kicad_pcb (формат S‑выражений KiCad) и извлекает информацию о цепях (net → список узлов ref/pin) и о футпринтах (ref → позиция, угол, пады).  
- **Способ:** использует библиотеку `sexpdata` для разбора S‑выражений, стандартные `open()` для чтения файлов, математику для преобразования координат падов с учётом поворота.

### `units.py`  
**Константа перевода миллиметров в нанометры**
