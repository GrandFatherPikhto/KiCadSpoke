# `kicadspoke/kicad` – Адаптер для взаимодействия с KiCad через IPC

## Назначение

Модули в директории `kicad/` предоставляют унифицированный интерфейс для работы с платой в KiCad через библиотеку `kipy` (KiCad Python IPC). Они скрывают детали низкоуровневого протокола, обеспечивают устойчивость к сбоям связи (повторные попытки при ошибках «KiCad is busy» и при обрыве соединения), упрощают выполнение транзакций (commit/rollback) и предоставляют удобный доступ к объектам платы.

**Ключевые возможности адаптера:**

- Поиск компонентов (футпринтов), падов, зон, цепей, via.
- Чтение пользовательских полей компонентов (например, `Role` для автоматического подбора).
- Получение bounding box'ов объектов (для keepout и коллизий) в одном батч-запросе.
- Транзакционное обновление и создание объектов с автоматическим откатом при ошибках.
- Переворот компонентов на другую сторону (flip) через GUI-действие (единственный способ получить корректное зеркалирование).
- Чтение текущего выделения с учётом групп (Group).
- Удаление объектов по UUID (для реестра расстановки via).
- **Диагностика и обход известного бага KiCad #24966** – краха на первой записи IPC при открытом редакторе схем.

## `interfaces.py` – Абстрактный интерфейс

Определяет абстрактный базовый класс `IBoardAdapter`, который описывает контракт для всех адаптеров к плате. Это позволяет легко подменять реализацию (например, для тестирования с моками) и гарантирует наличие всех необходимых методов.

**Полный список методов интерфейса:**

| Метод | Описание |
|-------|----------|
| `refresh_board()` | Обновить данные платы из KiCad (перечитать все объекты). |
| `get_footprint(ref)` | Получить футпринт по референсу (например, `IC1`). |
| `get_footprints()` | Получить все футпринты платы. |
| `get_vias()` | Получить все существующие переходные отверстия (via). |
| `get_selected_items()` | Получить текущее выделение (с учётом групп). |
| `get_field_value(fp, field_name)` | Прочитать значение пользовательского поля компонента (например, `Role`). |
| `get_footprint_pads(fp)` | Получить площадки (пады) футпринта. |
| `get_pad_by_number(fp, number)` | Найти площадку по номеру (например, `'1'`, `'145'`). |
| `get_zone_by_name(name)` | Найти зону по имени (Rule Area). |
| `get_net_by_name(name)` | Найти цепь по имени. |
| `get_bounding_boxes(items)` | Получить ограничивающие прямоугольники (Box2) для списка объектов (батч-запрос). |
| `begin_commit()` | Начать транзакцию. |
| `push_commit(commit, description)` | Применить транзакцию. |
| `drop_commit(commit)` | Откатить транзакцию. |
| `update_items(items)` | Обновить существующие объекты на плате. |
| `create_items(items)` | Создать новые объекты на плате. |
| `flip_selected(footprints)` | Перевернуть выделенные футпринты (зеркалирование на другую сторону). |
| `commit_with_retry(description, work_fn, retries)` | Выполнить работу в транзакции с автоматическим повторением при ошибках IPC. |
| `create_via(position, net, drill_mm, diameter_mm)` | Создать объект `Via` (без добавления на плату). |
| `remove_by_id(uuid_str)` | Удалить объект по UUID (для реестра). |

## `adapter.py` – Реализация `KiCadBoardAdapter`

Это основная реализация интерфейса, инкапсулирующая все вызовы к `kipy`. Она добавляет внутреннюю логику для надёжности, диагностики и удобства.

### Ключевые особенности

#### 1. Устойчивость к сбоям IPC

- **Ретраи при «KiCad is busy»** – все мутирующие операции обёрнуты в `_mutating_call()`, которая при получении `ApiError` с текстом "not ready" делает повторные попытки с увеличивающейся задержкой (до 3 попыток).
- **Обработка обрыва соединения** – при `ConnectionError` (обычно означает, что KiCad упал) выбрасывается понятное сообщение, указывающее на известный баг #24966 и предлагающее workaround.
- **`commit_with_retry`** – обёртка для произвольной работы в транзакции с автоматическим откатом и повторными попытками при любых исключениях.

#### 2. Обход бага KiCad #24966

Известный дефект KiCad 10.0.4: первая запись через IPC (`update_items` или `create_items`) может привести к краху всего процесса, если в сессии открыт редактор схем и не было ни одной интерактивной правки. Адаптер содержит:

- **`check_write_crash_risk()`** – вызывается перед первой мутирующей операцией. Проверяет через `get_open_documents()`, открыта ли схема, и если да – выдаёт предупреждение в лог с рекомендацией либо закрыть схему, либо сделать интерактивную правку в PCB.
- **Обёртка `_mutating_call()`** – перехватывает `ConnectionError` и логирует его как вероятное падение KiCad, давая пользователю конкретную инструкцию.

Таким образом, адаптер не предотвращает крах (это невозможно из клиента), но даёт чёткий диагноз и workaround, что сэкономило много часов отладки.

#### 3. Батч-запросы

Метод `get_bounding_boxes(items)` принимает список объектов и возвращает список `Box2` одним запросом к `board.get_item_bounding_box()`. Это значительно сокращает количество IPC-вызовов при построении keepout и проверке коллизий.

#### 4. Корректная работа с выделением и группами

`get_selected_items()` не просто возвращает `list(board.get_selection())`, а дополнительно разворачивает группы (`Group`), так как у них свойство `.items` всегда пустое, а реальные участники лежат в `.proto.items`. Для этого он сканирует все футпринты и via платы, сопоставляя их UUID с UUID в `proto.items` групп. Это критично для команды `extract` и для режима клонирования «по выделению».

#### 5. Чтение пользовательских полей

Метод `get_field_value(footprint, field_name)` фильтрует `footprint.texts_and_fields` по типу `Field` (а не просто по наличию атрибута `name`), так как там могут быть также объекты `BoardText` (обычный текст на шёлкографии). Это гарантирует, что мы читаем именно поле, добавленное в Eeschema.

#### 6. Флип через GUI-действие

`flip_selected(footprints)` использует `kicad.run_action("pcbnew.InteractiveEdit.flip")` – это единственный способ выполнить «настоящий» переворот компонента с зеркалированием площадок и шелкографии. Простое присвоение `fp.layer = BoardLayer.BL_B_Cu` меняет только слой, но не зеркалирует геометрию. После вызова необходимо перечитать плату через `refresh_board()`.

#### 7. Создание via

`create_via(position, net, drill_mm, diameter_mm)` создаёт объект `Via` с заполненными полями (тип, позиция, цепь, диаметры), но **не добавляет** его на плату – для этого требуется вызов `create_items([via])` внутри транзакции. Это разделение позволяет подготовить несколько via, а затем создать их одной транзакцией.

#### 8. Удаление по UUID

`remove_by_id(uuid_str)` используется реестром для удаления устаревших via. Создаёт объект `KIID` и вызывает `board.remove_items_by_id()`. Если объект с таким UUID уже не существует, метод возвращает `False` и логирует предупреждение, но не выбрасывает исключение – это позволяет gracefully обрабатывать ситуации, когда via была удалена вручную.

#### 9. Внутренние константы и настройки

- Таймаут по умолчанию – `DEFAULT_TIMEOUT_MS` из `constants.py` (20000 мс).
- В конструкторе можно переопределить таймаут параметром `timeout_ms`.
- Логирование ведётся через модуль `logging` с уровнем DEBUG/INFO.

## Примеры использования

### 1. Подключение и получение платы

```python
from kicadspoke.kicad.adapter import KiCadBoardAdapter

adapter = KiCadBoardAdapter(timeout_ms=30000)
adapter.refresh_board()
fp = adapter.get_footprint("IC1")
if fp is None:
    print("IC1 не найден")
```

### 2. Чтение поля Role

```python
role = adapter.get_field_value(fp, "Role")
print(f"Role: {role}")
```

### 3. Получение падов и их bbox

```python
pads = adapter.get_footprint_pads(fp)
bboxes = adapter.get_bounding_boxes(pads)
for pad, bbox in zip(pads, bboxes):
    if bbox:
        print(f"Pad {pad.number}: bbox {bbox.pos.x/1e6:.3f} x {bbox.pos.y/1e6:.3f} мм")
```

### 4. Транзакционное перемещение компонента

```python
fp = adapter.get_footprint("C5")
old_pos = fp.position
fp.position = Vector2.from_xy(old_pos.x + int(1.0 * MM), old_pos.y)
ok = adapter.commit_with_retry("Move C5", lambda: adapter.update_items([fp]))
if ok:
    print("Перемещение выполнено")
else:
    print("Ошибка")
```

### 5. Создание via с регистрацией

```python
net = adapter.get_net_by_name("GND")
via = adapter.create_via(position, net, drill_mm=0.3, diameter_mm=0.6)
adapter.commit_with_retry("Create via", lambda: adapter.create_items([via]))
```

### 6. Флип компонента

```python
fp = adapter.get_footprint("C6")
adapter.flip_selected([fp])
adapter.refresh_board()  # перечитываем объект
fp = adapter.get_footprint("C6")
print(f"Новый слой: {fp.layer}")
```

## Обработка ошибок и диагностика

### Известные проблемы

- **Крах KiCad на первой записи (#24966)** – если в сессии открыт редактор схем и не было интерактивных правок. Адаптер выдаёт предупреждение и логирует `ConnectionError` как вероятное падение.
- **«KiCad is busy»** – возникает, когда KiCad занят модальным диалогом или длительной операцией. Адаптер автоматически повторяет запрос до 3 раз с паузами.
- **Протухшие UUID** – при удалении via через `remove_by_id()`, если объект уже удалён, метод возвращает `False` и логирует предупреждение.

### Диагностические инструменты

Для отладки соединения и выявления крашей рекомендуется использовать диагностический скрипт `diagnose_first_write_crash.py` (см. `diagnostics.md`) и внешний охотник `hunt-proc.ps1` для перехвата дампов падений.

## Нестабильные и недокументированные API

В адаптере используются следующие элементы, которые выходят за пределы стабильного публичного API KiCad/kipy:

| Элемент | Статус | Причина |
|---------|--------|---------|
| `run_action("pcbnew.InteractiveEdit.flip")` | **Нестабильный** | GUI-действия могут меняться между версиями KiCad. |
| `Group.proto.items` | **Недокументированный** | Внутреннее поле protobuf, необходимое для получения участников группы. |
| `footprint.texts_and_fields` | **Недокументированный** | Используется для чтения пользовательских полей (`Field`). |
| `footprint.definition.items` | **Недокументированный** | Используется для получения падов компонента. |
| `board.remove_items_by_id()` | **Новый, потенциально нестабильный** | Добавлен недавно (июль 2025), может измениться. |
| `kicad.get_open_documents()` | **Официальный, но редко используемый** | Используется для проверки открытой схемы (предупреждение о краше). |

Все эти места **документированы** в коде и покрыты модульными тестами, что позволяет быстро обнаружить поломку при обновлении KiCad или `kipy`.

## Зависимости и требования

- `kipy` – официальная Python-библиотека для IPC KiCad (устанавливается отдельно, версия 0.7.1 или новее).
- `kipy.board_types` – типы данных KiCad (FootprintInstance, Pad, Via, Net, Field, Group, BoardLayer и др.).
- `kipy.geometry` – геометрические примитивы (Vector2, Box2, Angle).
- `kipy.errors.ApiError`, `kipy.errors.ConnectionError` – для обработки ошибок IPC.
- Стандартные пакеты Python: `logging`, `time`, `typing`.
