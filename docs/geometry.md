# `kicadspoke/geometry` – Геометрические утилиты

## Назначение

Модули в директории `geometry/` предоставляют низкоуровневые геометрические функции и классы, используемые для расчёта позиций компонентов и via, построения ограничивающих областей (keepout), поиска свободного места, предсказания положения падов после перемещения/поворота, генерации сетки термовиа и преобразования локальных координат шаблонов в глобальные.

Эти модули **не зависят** от KiCad и адаптера – они оперируют только координатами и векторами, что делает их легко тестируемыми и переиспользуемыми. Они используются в основном в `placement/services/manual_position_calculator.py`, `via_planner.py` и других модулях расстановки.

## Файлы и функции

### `keepout.py`

**Назначение:**  
Определяет класс `Rect` (выровненный по осям прямоугольник) и предоставляет функции для построения keepout-областей из bounding box'ов, проверки точки на занятость (с учётом радиуса via) и поиска свободного места вокруг идеальной позиции (по спирали или вдоль прямой).

**Основные классы и функции:**

| Имя | Описание |
|-----|----------|
| `Rect` | AABB-прямоугольник с методами `from_bbox` (из Box2), `from_circle` (аппроксимация окружности квадратом), `intersects`. |
| `point_is_clear(point, via_radius, keepout)` | Проверяет, свободна ли точка (окружность радиуса `via_radius` не пересекает ни один keepout-прямоугольник). |
| `build_keepout(bboxes, clearance_mm, mm_per_unit)` | Принимает список Box2 (от адаптера), строит список `Rect` с зазором `clearance_mm`. |
| `find_free_point(ideal, keepout, via_radius, ...)` | Ищет ближайшую свободную точку по спирали (кольцами). Учитывает предпочтительное направление. |
| `find_free_point_along_line(ideal, keepout, via_radius, line_direction, ...)` | Ищет свободную точку вдоль прямой (в обе стороны от `ideal`). |

**Используется в:** `via_planner.py` для поиска места под термовиа (единственный случай, где требуется автоматический поиск). Для остальных via поиск не применяется, так как они размещаются строго по координатам шаблона.

### `pad_projection.py`

**Назначение:**  
Предсказывает абсолютную позицию конкретного пада компонента, если компонент будет перемещён в новое место и повёрнут на заданный угол (с учётом возможного зеркалирования при флипе на обратную сторону). Логика централизована в этом модуле, чтобы избежать дублирования в разных частях проекта.

**Основные функции:**

| Имя | Описание |
|-----|----------|
| `local_pad_offset(fp, pad)` | Возвращает смещение пада относительно центра футпринта в **неповёрнутой** локальной системе координат (постоянная геометрия). |
| `predict_pad_position(fp, pad, dest, angle_deg, needs_flip)` | Предсказывает абсолютную позицию пада после перемещения футпринта в `dest` и поворота на `angle_deg` с учётом возможного зеркалирования при флипе (зеркалирование локального X). |

**Используется в:**  
В текущей версии `KiCadSpoke` эта функция **не используется**, так как все via (включая GND via) теперь вычисляются исключительно по геометрии шаблона, без обращения к живой плате. Однако модуль оставлен для потенциального использования в будущем (например, для диагностики или для других типов via).

---

### `spoke_layout.py`

**Назначение:**  
Преобразует локальные координаты шаблона спицы (`along`, `across`) в абсолютные координаты платы с учётом сдвига `(shift_x, shift_y)` и поворота `rotation_deg` конкретной спицы. Также вычисляет итоговый угол поворота компонента и генерирует все via (как на уровне спицы, так и на уровне компонента) в виде `ResolvedVia` с абсолютными координатами и цепью.

Этот модуль является **ключевым** для генерации всей геометрии расстановки.

**Основные классы и функции:**

| Имя | Описание |
|-----|----------|
| `rotate_local_offset(along_mm, across_mm, rotation_deg)` | Поворачивает локальный вектор `(along, across)` на заданный угол вокруг нуля (без переноса). |
| `local_to_absolute(origin, along_mm, across_mm, rotation_deg)` | Преобразует локальный вектор в абсолютную позицию относительно `origin` с учётом поворота. |
| `ResolvedVia` | Полностью разрешённая via: абсолютная позиция, цепь, параметры отверстия. |
| `ComponentLayout` | Структура, описывающая размещение одного компонента: ref, роль, позиция, угол, список via. |
| `SpokeLayout` | Структура, описывающая все элементы спицы: origin, список via (уровня спицы), список компонентов. |
| `apply_spoke_geometry(pad_position, spoke, template, rule_net, role_to_ref)` | Основная функция – принимает позицию пада FPGA, данные спицы (`ManualSpoke`), шаблон (`SpokeTemplate`), цепь правила и сопоставление роль→ref. Возвращает `SpokeLayout` с абсолютными координатами всех элементов. |

**Используется в:** `manual_position_calculator.py` для расчёта позиций компонентов и via.

### `thermal_grid.py`

**Назначение:**  
Вычисляет абсолютные координаты для массива термовиа под термопадом компонента (например, `IC1`). Учитывает размер пада, отступы от края, количество строк/столбцов и возможность шахматного порядка (stagger).

**Основные функции:**

| Имя | Описание |
|-----|----------|
| `get_pad_size(pad)` | Возвращает размеры медного слоя пада `(width, height)` в нанометрах. |
| `compute_thermal_via_grid(pad, rows, cols, margin_mm, stagger)` | Генерирует список абсолютных позиций для via, размещённых равномерно внутри площадки с учётом отступов `margin_mm`. |

**Используется в:** `via_planner.py` для планирования термовиа.

---

### `placement.py` – **УСТАРЕЛ**

**Назначение:**  
Содержит старую функцию `compute_position`, которая использовалась для автоматического размещения компонентов относительно границы зоны (`boundary`). После перехода на ручную стратегию (шаблоны) этот файл больше не используется и оставлен только для обратной совместимости со старыми тестами. В ближайшее время будет удалён.

### `__init__.py`

## Взаимосвязи с другими модулями

- `keepout.py` и `thermal_grid.py` используются в `placement/services/via_planner.py`.
- `pad_projection.py` не используется в текущей версии, но оставлен для будущих нужд.
- `spoke_layout.py` – ключевой модуль для новой архитектуры; используется в `placement/services/manual_position_calculator.py` и `via_planner.py` (для термовиа).

---

## Примечания

- Все координаты и размеры во всех функциях ожидаются в **нанометрах**, если не указано иное (поля `_mm` принимают миллиметры и конвертируются внутри с помощью `MM = 1_000_000` из `utils/units.py`).
- `find_free_point` и `find_free_point_along_line` сейчас используются только для термовиа, так как все остальные via размещаются строго по заданным координатам без поиска.
- Модуль `spoke_layout.py` полностью независим от зоны и `boundary` – вся геометрия задаётся в локальной системе шаблона, что делает конфигурацию поворотоинвариантной и предсказуемой.
